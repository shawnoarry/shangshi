# 工作流程与资料取用规则

> 更新日期：2026-05-25
> 用途：统一当前仓库的资料取用优先级、目录职责、依赖与 skill 状态，避免旧稿、灵感稿和执行稿互相覆盖。
> 参考方式：本规则借鉴 `F:\项目\机器人` 项目的工作区治理方法，只采用“口径优先级、目录定位、提交出口保护、过程归档”等技术规范，不引用其项目内容。

## 一、当前项目定位

本仓库服务上实集团成立45周年特别活动的资料整理、文稿策划和后续剧本工程。当前最紧急的写作对象是开场组合节目中的南洋烟草情景剧段落。

南洋烟草情景剧只是45周年大活动中的一个语言类/情景剧段落，不是整台活动总流程，也不是全部语言类节目的统一模板。后续如推进“上实小镇”、音乐剧串联、童声讲述或其他段落，应另建对应入口，不用南洋烟草资料区覆盖全部节目。

## 二、口径优先级

后续创作、改稿和判断冲突时，默认按以下顺序执行：

1. 用户最新明确意见、导演会最新口径、人工确认的提交版本。
2. `docs/01_会议纪要/20260523导演会会议记录工作汇总.md`。
3. `docs/02_执行稿/南洋烟草开场情景剧剧情大纲指导稿.md`。
4. `docs/05_文稿工程/01_剧本大纲/20260525_南洋烟草开场情景剧_剧情大纲_提交参考版.md` 及后续同目录新版本。
5. `docs/03_资料索引/` 中的时间轴、情节索引、素材池和分段概述。
6. `work/上实百廿.txt`、`work/上实百廿.pages.jsonl` 和 `画册扫描.pdf` 原页。
7. `docs/04_方案储备/` 中的母题和结构备选。
8. `docs/灵感素材库/` 中的早期灵感稿。

低优先级材料可以提供表达、标题、舞台意象或替换思路，但不能反向推翻更高优先级的口径。

## 三、目录职责

- `docs/00_项目说明/`：仓库使用、资料同步、依赖、流程和取用规则。
- `docs/01_会议纪要/`：会议原始转写、整理纪要、最新口径和任务拆解。
- `docs/02_执行稿/`：已经进入执行讨论的规划稿、指导稿和当前口径稿。
- `docs/03_资料索引/`：史实、时间轴、情节素材和页码导航。
- `docs/04_方案储备/`：结构备选、母题、表达方向和可替换方案。
- `docs/05_文稿工程/`：正式大纲、分场台本、提交稿和版本归档。
- `docs/06_新增资料/`：后续新增材料的原始文件、待整理摘录和已整理资料卡。
- `docs/灵感素材库/`：早期灵感稿、AI试写稿、弃用或降权结构。
- `过往项目格式案例/`：只看提交版大纲和成稿的格式、版式、栏目和文风，不取内容事实。
- `work/`：OCR文本、分页jsonl、可重新生成的中间成果。

## 四、文稿流转规则

新正式大纲、分场台本和对外流转稿，优先进入 `docs/05_文稿工程/`，不继续堆在 `docs/02_执行稿/`。

`docs/02_执行稿/` 保留已经形成的执行口径，主要作为后续文稿的依据；`docs/05_文稿工程/` 承接真正的写作、改稿和提交版本。

重要修改原则上先保留旧版本，再建立新版本。已经被降权但仍有参考价值的文档，不主动删除；若需要整理，优先移动到“灵感素材库”“方案储备”或“版本归档”，并在 README 或索引中说明取用边界。

## 五、史实核验流程

涉及年份、人名、金额、机构名、事件因果、引文和敏感历史表述时，按以下顺序核验：

1. 先看 `docs/03_资料索引/上实百廿标志性情节点时间轴.md` 判断时间关系。
2. 再看 `docs/03_资料索引/上实百廿全书情节索引.md` 和 `docs/03_资料索引/上实百廿候选节目素材池.md` 判断素材位置。
3. 用 `rg` 在 `work/上实百廿.txt` 中检索关键词。
4. 用 `work/上实百廿.pages.jsonl` 追溯页码。
5. 最终回到 `画册扫描.pdf` 原页确认。

OCR 文本适合检索，不等于最终史实。正式台词、提交大纲和导演阐述中如出现具体数字、引文或判断，应以原 PDF 页为准。

## 六、依赖与 skill 状态

当前仓库现有依赖很轻：

- Node.js / npm：用于运行 PDF 文本抽取和页面渲染脚本。
- `pdf-parse`：已写入 `package.json`，供 `scripts/extract-pdf-text.mjs` 和 `scripts/render-pdf-pages.mjs` 使用。
- PowerShell + Windows OCR：供 `scripts/ocr-windows-pages.ps1` 使用。
- Git LFS：用于同步根目录的大 PDF 原始资料。

本轮整理不需要新增 npm 依赖，也不需要安装新的 skill。`skills-lock.json` 中记录了 `pdf` skill，但当前仓库内未检测到 `skills/pdf/SKILL.md`，这不影响现有脚本运行。后续如果要把 PDF/OCR 流程做成固定 agent skill，可再通过 skill 安装工具补齐。

过往项目 `.docx` 样例目前只用于参考格式，读取格式骨架可用 PowerShell/OpenXML 直接完成，不需要新增依赖。若未来需要批量生成提交版 `.docx`，再考虑引入 `python-docx` 或复用已有 docx 格式转换工具。

## 七、过往项目格式案例使用规则

`过往项目格式案例/提交版大纲格式/` 主要参考：

- 标题写法。
- “核心主题/核心主旨”“内容梗概/剧情概述”“整体风格与结构”“角色”“剧情大纲”等栏目顺序。
- 段落颗粒度和对上提交时的说明语气。

`过往项目格式案例/提交版成稿格式/` 主要参考：

- 成稿版的角色行、舞台提示、台词排版。
- 多媒体、灯光、音乐提示如何写得可排练。

这些案例不参与南洋烟草事实判断，不替代《上实百廿》和导演会口径。

## 八、禁止事项

- 不用早期灵感稿覆盖当前执行稿。
- 不把南洋烟草资料区当成全部语言类节目的总资料区。
- 不在没有核验的情况下把 OCR 结果写成确定史实。
- 不直接删除旧方案、旧大纲和过往格式案例。
- 不把复杂历史政策段落强行写进开场情景剧结尾。
- 不把烟草题材处理成消费展示；烟标、广告和烟盒只作为历史物证和民族工业记忆使用。
