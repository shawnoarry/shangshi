# 文稿工程节目对照与流转索引

> 更新日期：2026-05-28
> 用途：作为 `docs/05_文稿工程/` 的跨节目总入口，说明各节目目录、当前基准和后续归位规则。

## 一、总原则

`docs/05_文稿工程/` 下所有节目均按“先分节目、再分流程”的方式建库。会议纪要、执行稿、资料索引、方案储备、新增资料、灵感归档、剧本大纲、分场台本、正式稿件、提交审核版和版本归档，都应优先归入对应节目目录。

后续如出现跨节目会议或资料，应先拆分为各节目摘录；确实无法拆分时，再建立跨节目统筹文件，并在相关节目目录中建立索引说明。

## 五、收到新文件时的归位规则

1. 导演会转写、会议纪要、反馈清单，进入 `00_会议纪要/`。
2. 甲方原始来稿、未改动底稿、旧定稿，进入 `01_执行稿/00_原始稿件/`。
3. 内部任务稿、需求拆解、修订口径，进入 `01_执行稿/`。
4. 资料整理、时间线、页码核验、采访索引，进入 `02_资料索引/` 或 `04_新增资料/`。
5. 替代创意、备用结构、未采用方案，进入 `03_方案储备/` 或 `05_灵感归档/`。
6. 故事梗概、结构版、分幕版，进入 `06_剧本大纲/`。
7. 正文台本、分场稿、导演展开稿，进入 `07_分场台本/`。
8. 排练稿、定稿、正式交付版，进入 `08_正式稿件/`。
9. 当前对上版本，进入 `09_提交审核版/`；旧审核版移入 `10_版本归档/`。

## 六、维护要求

1. 各节目目录的 `节目说明.md` 需保持为该节目的入口文件。
2. 各主文件夹中的 `本目录存放说明.md` 为后续归位的直接依据。
3. 某节目形成新审核基准后，应同步更新该节目 `节目说明.md` 和本索引。
4. 不使用没有节目限定的“最新版本”“当前稿”等模糊命名。
