# 工作流程与资料取用规则

> 更新日期：2026-05-28
> 用途：统一当前仓库的资料取用优先级、节目归档方式和后续文稿流转规则。

## 一、当前项目定位

本仓库服务上实集团成立45周年特别活动的资料整理、文稿策划和后续剧本工程。当前文稿工作按节目建库，核心入口为 `docs/05_文稿工程/`。

除仓库级说明和原始 OCR 工作成果外，会议纪要、执行稿、资料索引、方案储备、新增资料、灵感稿、提交稿和归档稿，都应进入对应节目目录。

## 二、目录职责

- `docs/00_项目说明/`：仓库使用、资料同步、依赖、流程和总规则。
- `docs/05_文稿工程/`：各节目文稿工程区。南烟、上药音乐剧新修、上实小镇、云南帮扶讲述等节目分别归档。
- `work/`：OCR 文本、分页 jsonl、可重新生成的中间成果。
- `过往项目格式案例/`：只参考格式、栏目、版式和文风，不取内容事实。

旧的顶层 `01_会议纪要`、`02_执行稿`、`03_资料索引`、`04_方案储备`、`06_新增资料`、`灵感素材库` 已改为节目内部分组，不再作为新文件入口。

## 三、节目内标准链路

后续新节目原则上使用以下目录模板：

```text
节目说明.md
00_会议纪要/
01_执行稿/
  └─ 00_原始稿件/
02_资料索引/
03_方案储备/
04_新增资料/
05_灵感归档/
06_剧本大纲/
07_分场台本/
08_正式稿件/
09_提交审核版/
10_版本归档/
```

## 四、口径优先级

后续创作、改稿和判断冲突时，先判断节目，再按以下顺序取用：

1. 用户最新明确意见、导演会最新口径、人工确认的提交版本。
2. 对应节目目录中的当前提交审核版或人工明确的审核基准文件。
3. 对应节目目录中的审核流程、编剧指导、演员角色规划、展开版说明。
4. 对应节目目录中的会议纪要、执行稿、资料索引、方案储备和新增资料。
5. OCR 文本、分页 jsonl 和原 PDF。
6. 早期灵感稿、弃用方案和降权结构。

低优先级材料可以提供表达、标题、舞台意象或替换思路，但不能反向推翻更高优先级口径。

## 五、文稿流转规则

1. 新材料先按节目归档，再按类型进入会议纪要、执行稿、原始稿件、资料索引、方案储备、新增资料、灵感归档、剧本大纲、分场台本、正式稿件、提交审核版或版本归档目录。
2. 跨节目会议或资料应尽量拆成各节目摘录；确实不能拆时，需在文件开头列明涉及节目，并在相关节目目录内建立索引。
3. 重要修改原则上先保留旧版本，再建立新版本。
4. 被降权但仍有参考价值的文档，移入对应节目的灵感归档、过程归档或版本归档。
5. 不使用没有节目限定的“当前最新大纲”“当前提交版”等说法。

## 六、史实核验流程

涉及年份、人名、金额、机构名、事件因果、引文和敏感历史表述时，按以下顺序核验：

1. 先看对应节目目录中的 `资料索引/`。
2. 再用 `rg` 检索 `work/` 中的 OCR 文本。
3. 用分页 jsonl 追溯页码。
4. 最终回到原 PDF 页面确认。

OCR 文本适合检索，不等于最终史实。正式台词、提交大纲和导演阐述中如出现具体数字、引文或判断，应以原 PDF 页为准。
