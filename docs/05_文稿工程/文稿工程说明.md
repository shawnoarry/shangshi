# 文稿工程说明

> 用途：说明 `docs/05_文稿工程/` 的节目化建库规则。
> 更新日期：2026-05-28

## 一、定位

`docs/05_文稿工程/` 是编剧团队负责节目的总工程区。当前文稿工作不再按“会议纪要/执行稿/资料索引”等公共顶层分类存放，而是先按节目建库，再在节目内部按流程分组。

仓库级规则保留在 `docs/00_项目说明/`。凡与具体节目直接相关的会议、原稿、资料、方案、大纲、台本、正式稿、提交稿和归档稿，均应进入对应节目目录。

## 二、当前节目目录

- `南烟剧目/`：南洋烟草情景剧《火种》及衍生版本，当前已形成完整项目链路。
- `上药音乐剧新修/`：上药音乐剧新修工作区，当前已纳入原版定稿和完整模板库。
- `上实小镇/`：上实小镇工作区，当前已搭建完整模板库，待后续文件归位。
- `云南帮扶讲述/`：云南帮扶讲述工作区，当前已搭建完整模板库，待后续文件归位。
- `节目对照与流转索引.md`：跨节目总入口和归位对照表。

## 三、标准节目结构

除个别节目因历史原因保留专项分支外，后续新节目原则上都按以下结构建库：

```text
节目说明.md
00_会议纪要/
01_执行稿/
  └─ 00_原始稿件/
02_资料索引/
03_方案储备/
04_新增资料/
05_灵感归档/
06_剧本大纲/
07_分场台本/
08_正式稿件/
09_提交审核版/
10_版本归档/
```

## 四、各分组职责

1. `00_会议纪要/`
   存放导演会、客户会、内部碰头会的逐字稿、会议纪要和反馈汇总。

2. `01_执行稿/`
   存放内部执行口径、任务拆解、工作底稿和早期工作文件。

3. `01_执行稿/00_原始稿件/`
   存放外部直接提供、原则上不应覆盖的原始 Word、PDF、扫描稿、旧定稿。

4. `02_资料索引/`
   存放史实核验、人物时间线、资料目录、页码线索和引用说明。

5. `03_方案储备/`
   存放尚未成为当前主线的替代方向、结构方案、题目备选和创意备份。

6. `04_新增资料/`
   存放后续新增网页资料、采访记录、转录文本、待整理素材和资料卡。

7. `05_灵感归档/`
   存放片段灵感、弃用段落、早期废稿和仍可回看的表达储备。

8. `06_剧本大纲/`
   存放剧情梗概、分幕结构、故事线梳理和展开版大纲。

9. `07_分场台本/`
   存放分场脚本、工作台本、带舞台提示的正文版本。

10. `08_正式稿件/`
    存放进入排练、内部确认或交付阶段的正式文本。

11. `09_提交审核版/`
    存放对上审核、客户流转、领导审看的当前有效版本。

12. `10_版本归档/`
    存放被替代的旧版本、旧提交稿和过程性留档文件。

## 五、归位规则

1. 新文件先判断节目，再判断类型，不再直接放入 `docs/` 顶层公共目录。
2. 外部原始来稿优先进入 `01_执行稿/00_原始稿件/`，不直接覆盖工作稿。
3. 已形成情节结构但尚未进入正文的文档，优先进入 `06_剧本大纲/`。
4. 已进入正文写作或分场阶段的文档，进入 `07_分场台本/` 或 `08_正式稿件/`。
5. 当前有效提交口径只保留在 `09_提交审核版/`；被替代版本移入 `10_版本归档/`。
6. 每个节目目录下的 `节目说明.md` 和各分组中的 `本目录存放说明.md`，共同作为后续归位依据。

## 六、命名建议

统一建议：

`YYYYMMDD_节目名_文件类型_主题或版本.ext`

示例：

- `20260528_上实小镇_剧情大纲_v0.1.md`
- `20260528_云南帮扶讲述_导演会会议纪要.md`
- `20260528_上药音乐剧新修_提交审核版_v1.docx`
